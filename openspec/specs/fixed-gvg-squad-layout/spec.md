## Purpose

Quy định workspace Bang Chiến linh hoạt với Đoàn và tổ đội động, xếp thành viên có constraint, cùng realtime refresh cho guild.

## Requirements

### Requirement: Dynamic division and squad inventory
The system SHALL allow a guild to have zero through five persisted Bang Chiến divisions. Each division SHALL contain zero through five squads. A guild owner SHALL create an empty division explicitly, and the system SHALL preserve that empty division until the owner deletes it. The system SHALL create squads explicitly inside a division and SHALL assign each new squad a unique immutable guild-scoped number generated by the server. Deleted squad numbers SHALL NOT be reused.

#### Scenario: A guild without a lineup is loaded
- **WHEN** an authorized user loads Bang Chiến for a guild with no division records
- **THEN** the system returns an empty division list without creating lineup data

#### Scenario: A guild owner creates an empty division
- **WHEN** the guild owner uses the add-division action while fewer than five divisions exist
- **THEN** the system saves an empty division and returns it in the lineup

#### Scenario: A guild owner reaches the division limit
- **WHEN** the guild already contains five divisions and the guild owner requests another division
- **THEN** the system rejects the request without changing the lineup

#### Scenario: A guild owner creates a squad
- **WHEN** the guild owner creates a squad in a division with fewer than five squads
- **THEN** the system creates a squad with six empty member slots and a server-generated immutable number

#### Scenario: A guild owner reaches squad capacity
- **WHEN** a division already contains five squads and the guild owner requests another squad
- **THEN** the system rejects the request without changing the lineup

#### Scenario: A squad is deleted
- **WHEN** the guild owner confirms deletion of a squad
- **THEN** the system permanently deletes that squad and its member-slot assignments while retaining the containing division

### Requirement: Division-tabbed squad workspace
The system SHALL render divisions in persisted order as Chrome-style tabs labeled only `Đoàn 1` through `Đoàn 5`. The workspace SHALL display exactly one selected division at a time and SHALL preserve an empty selected division. The tab strip SHALL provide an owner-only add-division action and SHALL not display squad counts in tab labels.

#### Scenario: A guild owner selects a division tab
- **WHEN** the guild owner selects `Đoàn 2`
- **THEN** the workspace displays only the persisted second division and marks `Đoàn 2` as selected

#### Scenario: A guild owner creates a division from the tab strip
- **WHEN** the guild owner selects the add-division control
- **THEN** the workspace creates and selects a new empty division tab

#### Scenario: An empty division is selected
- **WHEN** the selected division has zero squads
- **THEN** the workspace displays the empty division and an owner-only action to create a squad

#### Scenario: A selected division is deleted
- **WHEN** a saved or realtime layout update removes the selected division
- **THEN** the workspace selects an adjacent remaining division, or shows the empty-lineup state when no division remains

### Requirement: Squad members and metadata
The system SHALL allow each squad to contain six member slots. A member SHALL be assigned to no more than one squad in the same Bang Chiến layout and SHALL be active in the guild. Each squad SHALL have a persisted owner-editable name that defaults to `Tổ đội <number>` and SHALL keep that name when moved between divisions or when slots change.

#### Scenario: A guild owner assigns a member
- **WHEN** the guild owner changes a squad's six member-slot assignments
- **THEN** the system accepts only active unassigned guild members and persists the complete slot assignment for that squad

#### Scenario: A guild owner clears a squad
- **WHEN** the guild owner clears a squad
- **THEN** the system removes all member assignments while retaining the squad, name, number, and containing division

#### Scenario: A guild owner renames a squad
- **WHEN** the guild owner confirms a valid inline squad-name edit
- **THEN** the system persists the name while keeping the immutable squad number unchanged

#### Scenario: A non-owner views a squad name
- **WHEN** a non-owner views Bang Chiến
- **THEN** the system displays squad names without name-edit controls

### Requirement: Owner-only resource mutations and realtime persistence
The system SHALL require `view:guild` access to read the lineup and SHALL permit only the guild owner to create, delete, move, reorder, rename, clear, or assign divisions and squads. Every successful lineup resource mutation SHALL return the canonical complete lineup and publish `gvg_lineup_updated`.

#### Scenario: A non-owner attempts a lineup mutation
- **WHEN** a guild member without the owner role submits a lineup resource mutation
- **THEN** the system rejects the request without changing the saved lineup or publishing an update

#### Scenario: A subscribed client receives an update
- **WHEN** a subscribed client receives `gvg_lineup_updated` for its active guild
- **THEN** the client refreshes and renders the canonical current lineup, including empty divisions
